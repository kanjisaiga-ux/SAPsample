# 結合・受入テストシナリオ

## 承認

1. 1段階承認で最終承認され、コールバックが1回だけ呼ばれる。
2. 同順に2名設定し、片方の承認後にもう片方が`SKIPPED`になる。
3. 次順の承認者は前順完了まで一覧に表示されない。
4. 自分以外の`READY` work itemをURLから直接承認できない。
5. 却下後、残りwork itemが`CANCELLED`になる。
6. 元申請を修正して再申請すると新しいRequestUUIDになる。
7. `IN_APPROVAL`が存在する間は重複申請できない。
8. ルートマスタ変更が進行中申請へ影響しない。
9. 未登録CallbackIDを実行できない。

## 入出庫

1. 購買発注参照101入庫が正しい数量・単位で登録される。
2. 購買発注番号または明細なしの101が保存エラーになる。
3. 原価センタなしの201が保存エラーになる。
4. 承認前はSAP標準入出庫伝票が存在しない。
5. 最終承認時だけ標準伝票が登録される。
6. 標準APIエラー時に申請が`POSTING_ERROR`となり、エラー内容を保持する。
7. 同じ承認申請の再実行で標準伝票が二重登録されない。
8. 却下後の修正・再申請で最新内容が登録される。

## 画面

1. 入出庫List ReportからObject Pageを開き、ヘッダ・明細を登録できる。
2. 承認申請アクション後、提案済みルートを編集して申請できる。
3. 承認者のList Reportには自分の`READY`対象だけが表示される。
4. 詳細情報から元伝票を開き、戻る操作で承認画面へ戻る。
5. 申請中は編集、削除、再申請アクションが無効になる。
